Consistency is evil

The we've always done it this way complex.

It's such a strange mindset to me. This is how its been done so this is how we should keep doing it. A halfwit child can see the flaws in this logic, and yet, It's a common and popular opinion. Lets say theres a file like: 


// Some globals
var thisthing = 1;
var thatthing = 2;

// 500 more lines

// The only use of those globals
var dopefunc() {
    return thisthing + thatthing;
}

// More crap

Now the moment a developer opens this file he makes a descision, to either:
A) Add more globals, to be consistent.
or
B) Be a decent human being.
Yes, there are times when consistency is important, I'm not arguing that. But if the only reason you have for an action is consistency then you are probably making the wrong choice. It's such an obvious falsehood.
"We've always done it this way" is the same sort of reasoning as "Its good because its good" . AKA No reasoning at all.

What's even worse is that people use this reasoning to protect their insecurities. "Yes X is technically best practice, but in this file the pattern has been to do Z so we should stick with that" The annoying thing about arguments like this is the fact that more often than not the person is against X because they don't understand it, not because they truly think that consistently using a bad practice is a good idea.
